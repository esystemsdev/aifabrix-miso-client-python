- Keep files <=500 lines and methods <=20-30 lines where feasible.
- Keep Google-style docstrings for public methods and type hints for all touched functions.
- Update docs/changelog in the same implementation scope as contract changes.

## Contract Mapping (Old -> New for Python SDK)
- Session refresh semantics:
  - Old: `/api/v1/auth/refresh` may be treated as body-token refresh path in some helper usage.
  - New: session-oriented behavior does not depend on request-body `refreshToken`.
- Device refresh semantics:
  - Old: `/api/v1/auth/login/device/refresh` with body `refreshToken`.
  - New: unchanged and explicit.
- M2M semantics:
  - Old: `x-client-token` service-to-service flow isolated from user session flow.
  - New: unchanged and explicitly guarded.
